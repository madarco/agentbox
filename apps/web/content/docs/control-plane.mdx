By default the host relay runs on your laptop and performs every privileged action a box can't do itself — `git push`, opening PRs, leasing credentials. Turn the laptop off and those stop for your cloud boxes.

The **control plane** is an optional hosted service that takes over the *centralized* concerns so cloud boxes don't depend on your laptop:

- **git credentials** — a GitHub App leases short-lived (1-hour), single-repo tokens to boxes on demand. The plane never holds a long-lived PAT and never your repo contents; a compromised box yields at most a 1-hour, single-repo token.
- **permission state** — pending approvals live in the plane's database; boxes poll for the verdict and you answer from anywhere (CLI or the deployment).
- **registry / events / status** — one cross-machine view of every box.

It is the same `@agentbox/relay` core wrapped as a **Next.js + Postgres app**. The identical code deploys to **Vercel** (managed) or **self-hosts on a VPS** (Postgres + `next start` via docker-compose) — only the host differs. It is stateless per request and does **no host execution**: host-local actions (`cp`, `download`, `checkpoint`) stay on the laptop relay; cloud boxes push to GitHub directly with a leased token.

<Callout type="info" title="Cloud boxes only">Only **cloud** boxes (hetzner, vercel, e2b, daytona) get the laptop-off guarantee. Local **docker** boxes still need the laptop relay for host-local actions (they bind-mount the host `.git`), so a docker box on a sleeping laptop is simply offline.</Callout>

## Set up + deploy

One command creates the GitHub App the plane leases tokens with, deploys the plane, and points your CLI at it:

```bash
agentbox control-plane setup
```

It will:

1. Open GitHub to **create the App** (Repository permissions: **Contents** + **Pull requests**, Read & write) and write `~/.agentbox/control-plane/control-plane.env` (App id + base64 private key + a generated admin token, mode `0600`). The App name is autogenerated (`agentbox-<rand>`); override with `--name`.
2. **Deploy** the plane — pick **Vercel** or **Hetzner** (or skip), or pass `--deploy vercel|hetzner|none`.
3. **Point your CLI at it** (`relay.controlPlaneUrl`) and poll `/healthz`.
4. Open the App's **repo-selection page** so you can authorize the repos your boxes work in.

Both deploy targets build the plane **from GitHub** (`--repo`, default `madarco/agentbox`; `--ref`, default `main`) — so you don't need a monorepo checkout, and a globally-installed CLI can deploy.

### Vercel

`--deploy vercel` builds the plane **from GitHub** via the Vercel API — no local checkout. It works in tiers so you don't have to think about ownership:

1. If you don't own `--repo`, AgentBox **forks it to your account with `gh`** and deploys the fork.
2. It creates a Git-connected project (Root Directory `apps/hub`, Next.js), **auto-provisions Neon Postgres**, sets the App env, and builds `--ref` from GitHub.
3. If Vercel can't connect the repo via the API (its GitHub App isn't on your account yet), it falls back to the **Vercel Deploy Button** — your browser opens to clone the repo + install the App + provision Postgres, then AgentBox sets the secrets and redeploys.

Requires `agentbox vercel login` (and `gh auth login` for the auto-fork — without `gh`, the Deploy Button does the fork in-browser).

<Callout type="info" title="Boots before it's wired">The control plane is built to **boot even with no secrets set** — a fresh deploy answers `/healthz` (reporting what's configured) and never 500s; admin/leasing endpoints return a clean 503 until the env is set + redeployed. So the button flow (deploy bare → AgentBox sets secrets → redeploy) is never a broken state. Deploying the published repo means you deploy a fork; re-sync it (`gh repo sync`) or re-run with `--ref` for updates.</Callout>

### Hetzner

`--deploy hetzner` provisions a small VPS (cloud-init installs Docker and `git clone`s `--repo` at `--ref`), brings up the docker-compose (Postgres + the app + Caddy), and serves it over **HTTPS at `https://<ip>.sslip.io`** — Caddy auto-obtains a Let's Encrypt cert, so there's no DNS to configure. The firewall locks SSH to your egress IP and opens `:80`/`:443`. Needs `agentbox hetzner login`. No repo ownership required (the VPS clones the public repo).

### Self-host (docker-compose)

Skip the deploy step and run the written env anywhere:

```bash
cd apps/hub
docker compose --env-file ~/.agentbox/control-plane/control-plane.env up --build
# Postgres + the app on :8787
agentbox control-plane set-url http://<host>:8787
```

`set-url` writes [`relay.controlPlaneUrl`](/docs/configuration); `agentbox control-plane status` checks reachability.

## Authorize your repos

The App can only lease a token for repos it's installed on. Authorize the current project's repo:

```bash
agentbox control-plane add
```

You don't usually need to run it: when you launch `agentbox claude` / `codex` / `opencode` against a configured control plane, AgentBox checks whether the project's repo is authorized and, if not, prompts you to open the add-repo page (asked once per project; unattended runs only warn).

## Creating boxes from the plane

The plane can enqueue box creation (`POST /remote/boxes`, admin-bearer) and return a job id. A long-running **worker** drains that queue and provisions the boxes — it leases a token, clones the repo, and hands the checkout to the normal provider create path:

```bash
# Run alongside the plane (self-host) or as a scheduled job; needs the App key + provider creds
agentbox control-plane worker --store "$POSTGRES_URL"
agentbox control-plane worker --store "$POSTGRES_URL" --once   # drain once and exit
```

The worker provisions **cloud** boxes only.

## Security

- Leased tokens are per-repo, minimal permissions (contents + pull requests), 1-hour, and never persisted (in-memory cache only). The plane holds only the App private key.
- A GitHub installation token is repo-scoped, not branch-scoped, so the lease gate auto-approves only `agentbox/*` branches; any other branch needs approval. Branch protection on protected branches is the real backstop.
- The plane gates `/admin/*` and `/remote/*` on a constant-time admin bearer, fail-closed — it refuses to boot without one.

See [sync and git](/docs/sync-and-git) for the laptop-relay credential model and the cloud providers ([hetzner](/docs/hetzner), [vercel](/docs/vercel), [e2b](/docs/e2b), [daytona](/docs/daytona)).
